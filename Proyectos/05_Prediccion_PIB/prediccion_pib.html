Contexto del proyecto

El Producto Interno Bruto (PIB) es la suma del valor monetario de todos los bienes y servicios producidos dentro de un país durante un período determinado. Su análisis permite conocer la evolución de la actividad económica, identificar tendencias y anticipar comportamientos futuros.

Este proyecto fue desarrollado como Servicio Social en la UAM Cuajimalpa (junio 2021 – mayo 2023), con el objetivo de construir modelos de aprendizaje automático capaces de pronosticar el crecimiento económico de México usando datos socioeconómicos del INEGI.

Objetivo general

Construir una base de datos con variables socioeconómicas del INEGI que presenten alta correlación con el PIB, y aplicar algoritmos de aprendizaje automático para realizar pronósticos del comportamiento económico de México en años posteriores al período de análisis.

Objetivos específicos

  • Seleccionar fuentes de datos del INEGI con relación directa al PIB (educación, empleo, comercio exterior, vivienda, inversión)
  • Limpiar e interpolar valores ausentes mediante regresión polinomial y promedios
  • Calcular el coeficiente de correlación de Pearson entre cada variable y el PIB
  • Construir una base de datos consolidada con las variables de mayor correlación
  • Implementar y comparar medias móviles, ARIMA y redes neuronales artificiales en R
Período de análisis: 2005–2020  ·  Fuente de datos: INEGI  ·  Tasa libre de riesgo de referencia: datos anuales  ·  Supervisora: Dra. Alma Rocío Sagaceta Mejía

Fuentes de datos y limpieza

Se seleccionaron 9 bases de datos del INEGI que cubren distintos sectores de la economía mexicana. Todas presentaban valores ausentes para algunos años del período 2005–2020.

Bases de datos utilizadas

  • Servicios de salud — establecimientos por tipo a nivel nacional
  • Población y vivienda — densidad, servicios, bienes por hogar
  • Ocupación nacional de empleo — empleo formal e informal por estado y género
  • Comercio exterior — exportaciones e importaciones por sector
  • Balanza comercial — industria manufacturera y mercancías
  • Educación — escolaridad, analfabetismo, niveles educativos por estado
  • PIB — actividades primarias, secundarias y terciarias (valores constantes 2013)
  • Exportaciones — por sector y período trimestral
  • Inversión fija bruta — construcción, maquinaria y equipo

Metodología de limpieza

Para los valores ausentes se aplicaron dos estrategias según la cantidad de datos faltantes:

Interpolación por promedio (pocos valores ausentes) $$\hat{y}_t = \frac{y_{t-1} + y_{t+1}}{2}$$
Interpolación por regresión polinomial de grado k $$\hat{y}_t = \beta_0 + \beta_1 t + \beta_2 t^2 + \cdots + \beta_k t^k + \varepsilon_t$$

Selección de variables por correlación de Pearson

De las 47 variables originales se calculó el coeficiente de correlación con el PIB (Variable 44). Se seleccionaron las 16 variables con mayor correlación para construir la base de datos final utilizada en los modelos de aprendizaje automático.

Coeficiente de correlación de Pearson $$r = \frac{\sigma_{xy}}{\sigma_x \, \sigma_y} = \frac{\sum_{i=1}^{n}(x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum_{i=1}^{n}(x_i-\bar{x})^2}\;\sqrt{\sum_{i=1}^{n}(y_i-\bar{y})^2}}$$

Fundamentos matemáticos

1. Medias móviles

Las medias móviles calculan un promedio ponderado de observaciones pasadas para estimar el siguiente valor de la serie. Se implementaron tres variantes:

Media móvil simple (SMA) — n períodos $$\hat{Y}_t = \frac{Y_{t-1} + Y_{t-2} + \cdots + Y_{t-n}}{n} = \frac{1}{n}\sum_{k=1}^{n} Y_{t-k}$$
Media móvil exponencial (EMA) $$\text{EMA}_t = Y_t \cdot K + \text{EMA}_{t-1} \cdot (1-K), \qquad K = \frac{2}{N+1}$$
Media móvil ponderada (WMA) $$\hat{Y}_t = \frac{\sum_{k=1}^{t} W_k \cdot Y_k}{\sum_{k=1}^{t} W_k}$$

2. Series de tiempo — Prueba de Dickey-Fuller

Antes de ajustar un modelo ARIMA es necesario verificar la estacionariedad de la serie. Una serie estacionaria satisface:

Condiciones de estacionariedad $$\mathbb{E}[Y_t] = \mathbb{E}[Y_{t+m}] \quad \forall\, t,m$$ $$\text{Var}(Y_t) = \text{Var}(Y_{t+m}) \quad \forall\, t,m$$ $$\text{Cov}(Y_t, Y_{t+m}) = \text{Cov}(Y_{t+m}, Y_{t+m+k}) \quad \forall\, t,m,k$$

La prueba de Dickey-Fuller contrasta la existencia de una raíz unitaria en el polinomio característico de la serie:

Hipótesis de la prueba Dickey-Fuller $$H_0: \lambda = 1 \quad \text{(la serie NO es estacionaria)}$$ $$H_a: \lambda < 1 \quad \text{(la serie ES estacionaria)}$$
Operador de diferencias (para obtener estacionariedad) $$\nabla Y_t = Y_t - Y_{t-1} \qquad \nabla^2 Y_t = Y_t - 2Y_{t-1} + Y_{t-2}$$

3. Transformaciones de datos

En modelos multivariados es fundamental que todas las variables compartan la misma escala:

Escalado min-max — transforma al intervalo [0, 1] $$Y_{\text{scaled}} = \frac{Y_i - Y_{\min}}{Y_{\max} - Y_{\min}}$$
Normalización estándar — media 0, desviación estándar 1 $$Y_{\text{norm}} = \frac{Y_i - \bar{Y}}{\sigma_Y}$$
Reescalado inverso (para pronóstico en escala original) $$Y_{\text{rescaled}} = Y_{\text{scaled}} \cdot Y_{\max}$$

4. Redes neuronales — Perceptrón

La unidad básica de una red neuronal es el perceptrón, que calcula una combinación lineal de sus entradas y aplica una función de activación no lineal:

Función de salida del perceptrón $$y = f\!\left(\sum_{i=1}^{n} X_i W_i + b\right)$$

Donde $X_i$ son las entradas, $W_i$ los pesos de conexión, $b$ el sesgo (bias) y $f(\cdot)$ la función de activación. Se evaluaron funciones sigmoidea, tangente hiperbólica y ReLU.

Error cuadrático medio (ECM) — criterio de selección del modelo $$\text{ECM} = \frac{1}{n}\sum_{i=1}^{n}\left(\hat{Y}_i - Y_i\right)^2$$

Implementación en R

Todo el análisis fue desarrollado en R. A continuación se presentan los fragmentos principales.

1. Limpieza e interpolación de datos

R
# Cargar librerías
library(readxl)
library(dplyr)
library(ggplot2)

# Cargar base de datos del INEGI
datos <- read_excel("base_inegi.xlsx", sheet = 1)

# Detectar valores ausentes
valores_na <- colSums(is.na(datos))
cat("Variables con valores ausentes:\n")
print(valores_na[valores_na > 0])

# Interpolación por promedio (para 1-2 valores ausentes)
interpolar_promedio <- function(vec) {
  for (i in 2:(length(vec) - 1)) {
    if (is.na(vec[i])) {
      vec[i] <- (vec[i - 1] + vec[i + 1]) / 2
    }
  }
  return(vec)
}

# Interpolación por regresión polinomial (para más valores ausentes)
interpolar_polinomial <- function(vec, grado = 2) {
  idx <- which(!is.na(vec))
  t   <- 1:length(vec)
  modelo <- lm(vec[idx] ~ poly(t[idx], grado))
  vec[is.na(vec)] <- predict(modelo, newdata = data.frame(t = t[is.na(vec)]))
  return(vec)
}

# Aplicar interpolación a todas las columnas
datos_limpios <- datos %>%
  mutate(across(everything(), ~ {
    n_na <- sum(is.na(.))
    if (n_na == 0) .
    else if (n_na <= 2) interpolar_promedio(.)
    else interpolar_polinomial(., grado = 2)
  }))

library(forecast)
library(tseries)

pib <- ts(datos_limpios$Variable_44, start = 2000, frequency = 1)

# ── Medias móviles ──────────────────────────────────────────────────────
ma3 <- ma(pib, order = 3)

4. Redes neuronales artificiales

R
library(neuralnet)

# ── Escalado de datos ────────────────────────────────────────────────────
escalar <- function(x) (x - min(x)) / (max(x) - min(x))
reescalar <- function(x_scaled, x_orig) x_scaled * max(x_orig)

base_escalada <- as.data.frame(lapply(base_final, escalar))

# ── Construcción de 10 redes neuronales ─────────────────────────────────
arquitecturas <- list(
  c(2, 2),       # Red 1
  c(2, 2, 2),    # Red 2
  c(4, 2, 2),    # Red 3  ← menor ECM
  c(6, 3, 2),    # Red 4
  c(2, 5, 4),    # Red 5
  c(4, 4, 4),    # Red 6
  c(5, 5, 5),    # Red 7
  c(6, 7, 6),    # Red 8
  c(10,10,10,10),# Red 9
  c(7,10,9,2,1)  # Red 10
)

formula_rn <- as.formula(
  paste("Variable_44 ~", paste(vars_seleccionadas, collapse = " + "))
)

# Entrenar todas las redes y calcular ECM
resultados_ecm <- data.frame(Red = character(), ECM = numeric())

for (i in seq_along(arquitecturas)) {
  set.seed(42)
  red <- neuralnet(
    formula    = formula_rn,
    data       = base_escalada,
    hidden     = arquitecturas[[i]],
    linear.output = TRUE,
    threshold  = 0.01
  )
  
  pred_scaled <- compute(red, base_escalada[, vars_seleccionadas])$net.result
  pred_orig   <- reescalar(pred_scaled, base_final$Variable_44)
  real_orig   <- base_final$Variable_44
  
  ecm <- mean((pred_orig - real_orig)^2)
  resultados_ecm <- rbind(resultados_ecm,
                          data.frame(Red = paste0("Red neuronal ", i), ECM = ecm))
  cat(sprintf("Red %2d — ECM: %.1f\n", i, ecm))
}

# Red con menor ECM
mejor_red <- resultados_ecm[which.min(resultados_ecm$ECM), ]
cat("\nMejor modelo:", mejor_red$Red, "— ECM:", mejor_red$ECM, "\n")
# Resultado: Red neuronal 3 (capas: 4-2-2) con ECM = 452,918.8

Conclusiones

De los modelos implementados, las redes neuronales resultaron las más adecuadas para pronosticar el PIB de México. La ventaja principal sobre medias móviles y ARIMA es su capacidad para trabajar con múltiples variables predictoras simultáneamente, permitiendo capturar la naturaleza multivariada del PIB.

Hallazgos principales

  • El PIB de México no presenta estacionariedad (p-value = 0.8396 en Dickey-Fuller), lo que limita el uso directo de modelos ARIMA clásicos.
  • La función auto.arima seleccionó un ARIMA(0,1,0) como mejor modelo univariado — equivalente a un modelo de caminata aleatoria con drift.
  • La Red neuronal 3 con arquitectura [4,2,2] y 16 variables predictoras obtuvo el menor ECM (452,918) y pronosticó $17,991,199 millones para 2021, con un error del 0.97% respecto al valor real ($17,817,973).
  • El 60–70% del tiempo del proyecto se dedicó a limpieza y estructuración de datos, confirmando que la calidad del dato es más determinante que el algoritmo elegido.

Limitaciones

  • Solo se contaba con datos anuales (20 observaciones) — insuficientes para separar conjuntos de entrenamiento y prueba de forma robusta
  • Los errores de la red no siguen distribución normal, afectando las pruebas de hipótesis
  • Usar datos trimestrales mejoraría significativamente la cantidad de observaciones disponibles

Extensiones posibles

  • Incorporar datos trimestrales del INEGI/BANXICO para aumentar el tamaño de muestra
  • Explorar modelos LSTM que capturen dependencias temporales de largo plazo
  • Comparar con modelos de árbol de decisión (Random Forest, XGBoost)
  • Incluir variables macroeconómicas externas (tipo de cambio, precio del petróleo)
